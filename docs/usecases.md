What you can do with this project, with pointers to the rest of the docs.

## 1. Reproduce norm emergence in a small society

Run a single society and watch a shared norm emerge. See the [CLI](cli.md) for all `run` flags and the [LLM environment variables](cli.md#llm-environment-variables).

```bash
export OLLAMA_MODEL=llama3.2:latest
cargo run --release -- run --population 10 --entrepreneurs 3 --network ws --ws-k 4 --ws-beta 0.1 --rounds 48 --seed 42
uv run crsec-tools visualize
```

You should see the adoption rate climb toward 1.0 and the conflict count rise then fall — see [Visualization](visualization.md).

## 2. Run fully offline (no live LLM)

The mock smoke exercises the entire pipeline (mechanisms → metrics → CSV/JSON outputs → `latest` symlink) with a scripted client, so no Ollama/OpenAI is needed:

```bash
cargo run --release --example mock_smoke -- results
uv run crsec-tools visualize
```

This is also what the integration tests use (`cargo test`), so the deterministic core is verified without any network.

## 3. Sensitivity analysis (population × network density)

Sweep the population and the WS rewiring probability to see when norms emerge and how fast:

```bash
cargo run --release -- sweep --population-values 6,10,20 --ws-beta-min 0.0 --ws-beta-max 0.5 --ws-beta-step 0.1 --runs 3 --seed 42
uv run crsec-tools visualize-sweep
```

See [Visualization](visualization.md) for the heatmaps and curves.

## 4. Inspect exactly what a run talked to

```bash
uv run crsec-tools show-experiment-settings --results-dir results/latest
```

This prints the config plus the LLM metadata (model, endpoint, temperature, seed, cache-hit rate, convergence, time-to-emergence) from `run_metadata.json`.
