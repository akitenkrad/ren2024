## 1. 小規模社会での規範創発を再現する

1 つの社会を実行し共有規範の創発を観察する．`run` の全フラグと [LLM 環境変数](cli.ja.md#llm-環境変数) は [CLI](cli.ja.md) を参照．

```bash
export OLLAMA_MODEL=llama3.2:latest
cargo run --release -- run --population 10 --entrepreneurs 3 --network ws --ws-k 4 --ws-beta 0.1 --rounds 48 --seed 42
uv run crsec-tools visualize
```

採用率が 1.0 へ上昇し，衝突数が初期急増→減少するさまが見られるはず — [可視化](visualization.ja.md) 参照．

## 2. 完全オフライン実行（live LLM 不要）

mock スモークはスクリプト化クライアントでパイプライン全体（メカニズム → 指標 → CSV/JSON 出力 → `latest` シンボリックリンク）を動かすため Ollama/OpenAI 不要:

```bash
cargo run --release --example mock_smoke -- results
uv run crsec-tools visualize
```

統合テスト（`cargo test`）もこれを使うため，決定論コアはネットワークなしで検証される．

## 3. 感度分析（人口 × ネットワーク密度）

人口と WS 再配線確率を走査し，規範がいつ・どれだけ速く創発するかを見る:

```bash
cargo run --release -- sweep --population-values 6,10,20 --ws-beta-min 0.0 --ws-beta-max 0.5 --ws-beta-step 0.1 --runs 3 --seed 42
uv run crsec-tools visualize-sweep
```

ヒートマップ・折れ線は [可視化](visualization.ja.md) を参照．

## 4. 実行が何と通信したかを確認する

```bash
uv run crsec-tools show-experiment-settings --results-dir results/latest
```

設定に加え `run_metadata.json` の LLM メタ（モデル・endpoint・温度・seed・cache-hit 率・収束・創発時刻）を表示する．
